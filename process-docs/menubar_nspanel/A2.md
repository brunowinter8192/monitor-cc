# NSPanel Refactor — Phase A.2 Build Narrative

## What Was Built

`dev/menubar_nspanel/p1_nspanel_probe.py` — standalone NSPanel menubar probe.

Key structural choices made during build vs. sketch:

**`_PanelController` ObjC target:** `NSObject` subclass with `togglePanel_(self, sender)` method. Button action wired as `btn.setAction_(b'togglePanel:')`. `togglePanel_` checks `app._panel_open` (Python bool) rather than `panel.isVisible()` — avoids round-tripping into ObjC for a state query we already track.

**`_make_nspanel()` returns `(panel, tv)` tuple:** PyObjC NSPanel is an ObjC-bridged object; arbitrary Python attributes cannot be set on it (`panel._probe_textview = tv` raises `AttributeError: 'NSPanel' object has no attribute '_probe_textview'`). Fix: return both objects from the factory, store on the Python app instance as `self._panel, self._panel_tv`. The `__init__` unpacks the tuple.

**`_update_panel_text(tv, sessions)`:** Takes `NSTextView` directly (not the panel), builds a full NSAttributedString with Menlo font and replaces `tv.textStorage()` each tick. Full replace is fine — `textStorage().setAttributedString_()` is atomic and flicker-free.

**`sys.path.insert` at module top:** `src.menubar.discover` is imported for live session data. Running from `dev/menubar_nspanel/` requires the project root on sys.path. Pattern taken from `dev/display/test_hover_map.py:13`.

**`_panel_open` tracks open/close state:** Checked in `togglePanel_` rather than calling `panel.isVisible()`. Consistent with how `_menu_open` works in production.

**NSMenu null + button wiring in `_tick` lazy-init:** Same pattern as production `_MenuDelegate` lazy-init. `self._nsapp` is not available in `__init__` — only after `app.run()` starts the AppKit loop and `initializeStatusBar()` completes (rumps `run()` line 1350). Guard: `if not self._initialized: try: ... except AttributeError: return`.

## Surprises

One unexpected issue during build: PyObjC NSPanel cannot carry Python attributes. The initial design assigned `panel._probe_textview = tv` which raised `AttributeError` on first launch. Fixed by returning `(panel, tv)` tuple from `_make_nspanel()`. Standard PyObjC constraint — ObjC bridged objects have fixed slot sets.

## Decision-Next

Phase B: port to `src/menubar/menubar.py`. Key changes vs. probe:
- Replace `NSTextView` with proper `NSScrollView + NSStackView` (or per-row `NSTextField` labels with click callbacks for `_focus_session`)
- Port `_rebuild_menu` → `_rebuild_panel` (full content rebuild when panel closed)
- Port `_update_menu_inplace` → `_update_panel_inplace` (per-row label update while panel open; `_panel_open` gate in `_tick`)
- Remove `_MenuDelegate`, `NSRunLoopCommonModes` import, `NSTimer` tracking-mode infrastructure
- Keep `_menu_open` rename to `_panel_open`, same gate semantics
- Add Quit item row in panel (currently via `rumps.quit_button` which is menu-attached — need alternative)
